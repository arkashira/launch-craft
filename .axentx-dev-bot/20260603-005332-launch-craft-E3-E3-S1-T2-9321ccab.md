Feature: **Launch‑Craft Automated Pre‑Flight Checklist**

**Description**  
The Launch‑Craft Automated Pre‑Flight Checklist (APFC) ensures every launch‑craft (e.g., rockets, drones, or high‑altitude balloons) completes a verified, repeatable set of pre‑flight steps before mission start. The feature integrates with existing telemetry, configuration, and safety systems to automatically gather status data, run validation rules, and present a concise, actionable checklist to operators. Only when all checklist items pass can the launch‑craft be cleared for countdown, reducing human error and increasing launch reliability.

**Implementation**

1. **Data Ingestion Layer**  
   * Connect to existing telemetry streams (e.g., sensor bus, CAN, MQTT) and configuration repositories (Git, DB).  
   * Normalise data into a unified schema (`craft_id`, `timestamp`, `parameter`, `value`, `status`).  

2. **Checklist Definition Engine**  
   * Store checklist templates per craft type in a version‑controlled JSON/YAML store.  
   * Each item includes:  
     - `id`  
     - `description`  
     - `source` (telemetry key or config file)  
     - `validation_rule` (Python expression or compiled rule)  
     - `severity` (blocker, warning, info)  

3. **Real‑Time Validation Service**  
   * Pull latest data, evaluate each rule, and produce a status (`PASS`, `FAIL`, `WARN`).  
   * Persist results to a fast key‑value store (Redis) for low‑latency UI access.  

4. **Operator UI Dashboard**  
   * Web‑based React dashboard showing the checklist per craft, colour‑coded by severity.  
   * One‑click “Acknowledge” for warnings, and “Override” with mandatory justification for blockers.  

5. **Clear‑to‑Launch Gate**  
   * Expose a REST endpoint (`/craft/{id}/clear`) that returns `true` only if all blocker items are `PASS`.  
   * Integrate with existing launch sequencer to automatically halt countdown if the gate returns `false`.  

6. **Audit & Reporting**  
   * Log every checklist run, operator actions, and overrides to an immutable audit store (e.g., append‑only S3 + Athena).  
   * Generate daily/weekly compliance reports for safety audits.  

**Benefits**

* **Safety** – Guarantees critical pre‑flight conditions are verified before launch, reducing aborts and incidents.  
* **Consistency** – Standardises pre‑flight procedures across all craft types, eliminating ad‑hoc checklists.  
* **Efficiency** – Automates data collection and validation, cutting manual preparation time by ~40 %.  
* **Traceability** – Full audit trail supports regulatory compliance and post‑mission analysis.  

**Technical Requirements**

* **Integration** – APIs to existing telemetry brokers (Kafka, MQTT) and configuration services (GitLab, Consul).  
* **Scalability** – Stateless validation microservice; horizontally scalable behind a load balancer.  
* **Reliability** – Deploy in a high‑availability Kubernetes cluster with health checks and circuit‑breaker patterns.  
* **Security** – Role‑based access control (RBAC) for UI and API; all communications TLS‑encrypted; audit logging of overrides.

// productPublicView.test.js
import { renderProductPage, generateJsonLD } from '@/views/public/products/show';
import { getProductBySlug, getAggregatedReviews } from '@/services/productService';

jest.mock('@/services/productService');

describe('Public Product View - Unit', () => {
  const mockProduct = {
    slug: 'awesome-gadget',
    name: 'Awesome Gadget',
    description: 'The best gadget ever.',
    screenshots: ['https://cdn.example.com/img1.png'],
    rating: 4.5,
    reviewCount: 12,
  };

  const mockReviews = [
    { reviewer: 'Alice', rating: 5, comment: 'Love it!' },
    { reviewer: 'Bob', rating: 4, comment: 'Very good.' },
  ];

  beforeEach(() => {
    getProductBySlug.mockResolvedValue(mockProduct);
    getAggregatedReviews.mockResolvedValue(mockReviews);
  });

  test('renders HTML with product data', async () => {
    const html = await renderProductPage('awesome-gadget');
    expect(html).toContain(mockProduct.name);
    expect(html).toContain(mockProduct.description);
    expect(html).toContain('<img');
    expect(html).toContain('src="https://cdn.example.com/img1.png"');
  });

  test('generates correct JSON‑LD', () => {
    const jsonld = generateJsonLD(mockProduct, mockReviews);
    const data = JSON.parse(jsonld);
    expect(data['@type']).toBe('Product');
    expect(data.name).toBe(mockProduct.name);
    expect(data.description).toBe(mockProduct.description);
    expect(data.image).toEqual(mockProduct.screenshots);
    expect(data.aggregateRating.ratingValue).toBe(String(mockProduct.rating));
    expect(data.aggregateRating.reviewCount).toBe(String(mockProduct.reviewCount));
  });

  test('includes all reviews in rendered list', async () => {
    const html = await renderProductPage('awesome-gadget');
    mockReviews.forEach(r => {
      expect(html).toContain(r.reviewer);
      expect(html).toContain(`rating-${r.rating}`);
      expect(html).toContain(r.comment);
    });
  });

  test('does not require authentication', async () => {
    const response = await request(app).get('/p/awesome-gadget');
    expect(response.status).toBe(200);
    expect(response.headers['set-cookie']).toBeUndefined();
  });
});
