## 1. Acceptance Criteria
1. **Product creation** – A product can be persisted with the fields `name`, `url`, `short_description`, and `category`; all are required and must respect length limits (name ≤ 100 chars, URL ≤ 2048 chars, description ≤ 500 chars, category ≤ 50 chars).  
2. **Database schema** – The `products` table contains columns `id (PK)`, `name`, `url`, `short_description`, `category`, `created_at`, `updated_at`. All columns are non‑null except `short_description` which may be empty but not null.  
3. **Uniqueness** – The combination of `name` + `url` must be unique to prevent duplicate entries of the same product.  
4. **Directory selection** – A product can be associated with zero‑to‑many `DirectorySelection` records (join table `product_directory_selections`) referencing the 10 supported directories.  
5. **Submission status** – Upon product creation, a `Submission` record is automatically generated for each selected directory with `status = 'pending'`.  
6. **Validation errors** – Attempting to save a product with missing required fields or invalid URL format returns a descriptive validation error.  
7. **Migration safety** – Running the migration on an empty DB creates the `products` table without errors; re‑running on an existing DB is a no‑op (idempotent).
