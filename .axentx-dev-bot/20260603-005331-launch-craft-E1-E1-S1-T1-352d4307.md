3. **Uniqueness** – The combination of `name` + `url` must be unique to prevent duplicate entries of the same product.

5. **Submission status** – Upon product creation, a `Submission` record is automatically generated for each selected directory with `status = 'pending'`.  
6. **Validation errors** – Attempting to save a product with missing required fields or invalid URL format returns a descriptive validation error.  
7. **Migration safety** – Running the migration on an empty DB creates the `products` table without errors; re‑running on an existing DB is a no‑op (idempotent).
