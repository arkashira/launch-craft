5. **Validation** – The form must validate that:  
   - All required fields are present.  
   - The URL is a well‑formed HTTP/HTTPS URL.  
   - The selected directories exist in the `directories` table.  
6. **Error Handling** – If any API call fails, the corresponding `ProductSubmission` status must be set to **'Failed'** and an error message stored.  
7. **User Feedback** – After submission, the user sees a summary table listing each selected directory and its current status (`Pending`, `Failed`, or `Success`).
