| **Make defaults explicit & configurable** | Provide a `DEFAULT_METADATA` constant and expose CLI flags (`--name`, `--description`, …) so users can override any field. |
| **Guard against regressions** | Add a unit test that asserts the file size and required keys. |

2. **Directory Checkboxes** — Exactly 10 supported directories are listed; user may select any subset (zero or more).
3. **Submission Dispatch** — On valid submit, the controller sends a correctly formatted payload (per each directory's API spec or email template) for every selected directory.
4. **Immediate Pending Status** — A `ProductSubmission` record with `status: 'Pending'` is persisted for each selected directory within the same request cycle, before any async job runs.
5. **URL Validation** — Server rejects any `url` that is not a well-formed `http://` or `https://` URL; form re-renders with an error message.
6. **Failure Isolation** — If one directory API call fails, only that submission's status is set to `'Failed'`; others remain `Pending` or advance normally.
7. **Post-Submit Summary** — After redirect, the user sees a status table showing each selected directory and its current `status` value.
